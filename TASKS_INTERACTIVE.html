Core Tasks

Problem: note rows above true headers make previews hard to interpret.

Outcome: stable schema baseline for later transforms.

Detailed guidance
In the original file, the first two rows are title/info rows, and the actual header row is at index 2 (0-based), where you see names like BioProject No., Country of origin, isolation source, Species, and Res profile [EUCAMP2].

Implement a header-row selector and test:
  • Wrong choice: row 0 gives noisy names like Unnamed: 1.
  • Correct choice: row 2 gives meaningful analysis columns.
Also sanitize duplicate/blank header names safely.

Problem: helper/meta columns add noise.

Outcome: focused dataset for analysis and export.

Detailed guidance
Add multiselect to remove columns from preview and exported data. Make behavior explicit and consistent between UI and output file.

Dataset-grounded example of a column that is often useful to exclude in a cleaner view:
  • Alias (mostly empty)

Problem: multi-value cells reduce machine readability.

Outcome: one value concept per column (e.g., separate host and matrix fields) for reliable filtering and grouping.

Detailed guidance
Let users choose source column, delimiter, and output names. Handle uneven split lengths safely and document naming collision behavior.

Concrete dataset example:
  • source column: isolation source
  • input values: broiler, meat / broiler, cecum
  • delimiter: comma
  • output columns: host and matrix

Problem: code lists in one cell are hard to filter directly.

Outcome: explicit indicator columns (e.g., has_CIP, has_NAL) that enable reproducible filtering and quick summaries.

Detailed guidance
Detect codes in a selected source column and generate deterministic, collision-safe indicator columns. Decide whether to keep the original column.

Concrete dataset example:
  • source column: Res profile [EUCAMP2]
  • row value: CIP, ERY, NAL, TET
  • generated indicators: has_CIP=1, has_ERY=1, has_NAL=1, has_TET=1
  • for sensitive: all resistance indicators should be 0

Problem: users need scoped subsets quickly.

Outcome: interactive row-level inclusion/exclusion before export.

Detailed guidance
Implement a compact operator set (equals, contains, empty/not-empty, etc.) and show row-count feedback as rules are applied.

Concrete dataset examples:
  • Country of origin = Germany
  • Species contains Campylobacter coli
  • Res profile [EUCAMP2] contains CIP

Problem: repeated decisions drift across sessions.

Outcome: reproducible cleaning conventions.

Detailed guidance
Store rules for empty-like tokens, naming style, and default include/exclude behavior. Apply these conventions consistently in later tasks.

Example convention set for this file:
  • Trim whitespace after splitting isolation source
  • Indicator naming convention: has_<CODE>
  • Treat placeholder values like NaN and blanks as empty
